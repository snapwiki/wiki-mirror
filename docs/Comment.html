In programming, a comment is a piece of string that is not be parsed and read by the compiler, which means it will be ignored and not be run.

Comments in Snap!

In Snap!, comments can be added by right-clicking the background of the scripting area or a block then selecting "add comment". The latter automatically attaches the comment to the block. Comments can be also attached afterwards by dragging them into a block (which draws a yellow halo); attaching them to a {prototype hat::other}::define+ will make the contents the help dialog for the custom block.

Comments can also be resized horizontally (the vertical size dynamically stretches to fit all the lines) and collapsed to only fit a single line to an extent of the current width.

Those comments don't take the form of a block (meaning that it does not look like or act like a block), but blocks that do the job of being comments can be created as custom blocks.

// This is an example of a comment. Comments can be stray (on its own line like this one) or be attached to a block.

move (-10) steps // negative steps move backward

asdf confusing block::other // confusing blocks can be explained with the help of a comment.

Comments in JavaScript

In JavaScript, comments can be created by wrapping the string within /* and */. Single-line comments can be also created by following the string by //.

/* This is a comment in JavaScript.

It takes multiple lines. */

// A stray comment on its own line.

Console.log(1+2) // Logs 3 into the console

Usage

Comments are often used to describe confusing information for people reading the code and to remind oneself of something whenever writing the code.

Known as "commenting out"[citation needed], comments can also be used to efficiently stop a piece of code from being run while preserving it by wrapping them in comment syntax. This cannot be done in block languages like Snap!, though.
