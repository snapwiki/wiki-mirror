go to x: () y: ()
go to x: (0) y: (0)
CategoryMotion
TypeCommand

The go to x: (0) y: (0) block is a block in the Motion category that moves the sprite to the specified coordinates.

Examples

Making a sprite reset its position when the project is restarted:

when flag clicked
point in direction (90)
go to x: (0) y: (0)

Ask the user for a position:

when flag clicked
script variables ((x)) ((y)) @delInput @addInput :: other
ask [x position?] and wait
set [x V] to (answer)
ask [y position?] and wait
set [y V] to (answer)
go to x: (x) y: (y)

Workaround

If you do not want to use the go to x: (0) y: (0) block, then you can use the set x to (0) and set y to (0) blocks to replicate it, however this has a different effect with pen. If you want the same effect you can use the go to [ V] block with a coordinate list.

Definition

This default definition can be found in Snap! 10.0 with the Blocks all the way option turned on. Remove or delete the block immediately after the definition hat if you want the script to be in action.

{move ((steps # = 10)) steps :: motion} :: define+
<t> primitive [gotoXY V] :: other
go to (list (x) (y) @delInput @addInput)
