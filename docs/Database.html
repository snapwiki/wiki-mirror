Database is a library in Snap!. It consists of blocks that modify the data stored in JavaScript local storage, a method of storing data within browsers. The data is accessible throughout the Snap! website, so are all projects.

Blocks

store key: [] value: [] in browser::other: Assigns a value to a key in the storage. If the key is already present, overrides the value.

(stored data in browser::other): Reports a list of all the stored data. Each item is a key-value pair mapped to a two-item list.

remove key: [] in browser::other: Removes a key and its assigned value from the storage.

clear data in browser::other: Clears all the key-value pairs from the storage.

(get value from key: [] in browser::other): Reports the assigned value of the specified key.

Insights

As the data stored in the local storage is accessible by any project, don't use generic, common names for keys like "level", "score" or "money", as there is a likely probability that there may be a project that causes data conflicts and unintentionally change the data from another, ruining the experience as one uses a project. Make sure to add a distinctive feature (for example, adding a prefix like "gameName_score") that sets the keys apart from keys other projects might use.

It is recommended to avoid the clear data in browser::other block as this also clears keys used by other projects, degrading user experience for unwitting users.
