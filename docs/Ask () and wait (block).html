This page is about the primitive block. For the reporter counterpart found in the Menus library, see ask () (block).

ask () and wait
ask [what's your name?] and wait
CategorySensing
TypeCommand

ask () and wait is a command block and a sensing block. Whenever this block is run, the caller sprite asks a prompt specified in the input. A text field will appear at the bottom of the stage. The prompt will appear as a speech bubble with a Sensing-colored border in a similar fashion to the say blocks, unless it is run from the stage or a hidden sprite, which in this case the prompt appears above the text field. The answer one gives will be reported through the answer

If the prompt is a list, a menu will appear instead of a text field. The menu structure and properties depend on how lists are nested.

Example Uses

Getting to know the user:

ask [What's your name?] and wait
say (join [Hi ] (answer) [!] @delInput @verticalEllipsis @addInput) for (2) secs

For creating quizzes:

ask [What's 54 times 9?] and wait
if <(answer) = [486]> {
  say [Correct!]
}
